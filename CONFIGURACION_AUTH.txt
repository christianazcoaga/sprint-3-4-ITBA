CONFIGURACION DEL SISTEMA DE AUTENTICACION
===========================================

Se ha implementado exitosamente el sistema de autenticacion con las siguientes caracteristicas:

1. BACKEND (Ya implementado)
   - Registro de usuarios con hasheo de contrasenas usando bcrypt
   - Login con validacion de credenciales y generacion de JWT
   - Rutas en /api/users/register y /api/users/login

2. FRONTEND (Ya implementado)
   - Pagina de registro en /registro
   - Pagina de login en /login
   - Contexto de autenticacion para manejar el estado del usuario
   - Persistencia de sesion usando localStorage
   - Navbar actualizado con estado de autenticacion

CONFIGURACION REQUERIDA:
------------------------

IMPORTANTE: Debes crear un archivo .env en la carpeta backend/ con las siguientes variables:

MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/
DB_NAME=nombre_de_tu_base_de_datos
JWT_SECRET=tu_clave_secreta_segura_aqui
PORT=3000

NOTA: El JWT_SECRET debe ser una cadena segura y aleatoria. Puedes generar una usando:
      node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

COMO USAR:
----------

1. Asegurate de que el servidor backend este corriendo (npm run dev en backend/)
2. Asegurate de que el cliente este corriendo (npm run dev en client/)
3. Navega a http://localhost:5173/registro para crear una cuenta
4. Navega a http://localhost:5173/login para iniciar sesion
5. El token JWT se almacena automaticamente en localStorage
6. La sesion persiste a traves de recargas de pagina
7. Usa el boton "Cerrar Sesion" en el navbar para salir
